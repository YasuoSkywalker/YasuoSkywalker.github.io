Qt学习记录-信号和槽机制

当某个时间发生后，比如某个按钮被点击了一下，它就会广播一个信号（signal）,如果某个对象对这个信号感兴趣，那么可以使用链接函数（connect）订阅这个信号，当收到信号时，被链接的槽函数（slot）会自动被回调，这就是信号和槽机制。

语法：

  • connect(信号发出者（对象），&信号（本质是一个函数，地址传递），信号接收者，槽函数（是一个函数，地址传递）)

槽发生重载的解决

有时候，当槽函数发生重载时，传统的connect写法Qt不知道该调用哪一个，这个时候要用到函数指针来解决问题。

void(类作用域下的一个指针)（明确参数）= 对应类作用域下的函数地址

带参数的信号和槽中参数的转递问题

信号函数在广播信号时，会连带其参数列表（类型，顺序）一起发出，让槽函数接收到。由此，信号函数的参数和槽函数的参数顺序，类型要对应，且信号函数的参数个数不能少于槽函数的参数个数

lambda表达式解决函数调用问题

lambda表达式相当于一个匿名函数

格式：

[ 模式 ] {参数} ->返回值类型 { 函数主体 }

其中模式填 = 表示值传递，& 表示引用传递，这里饥馑是为了让lambda表达式能识别函数体内的变量，一般采用 =
